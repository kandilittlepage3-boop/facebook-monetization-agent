Reporting a Vulnerability
We take the security of our project seriously. If you believe you have found a security vulnerability, please do not report it through the public issue tracker.
Instead, please report it via one of the following methods:

GitHub Security Advisory: Use the "Report a vulnerability" button on the repository's Security tab.
Email: Send a detailed report to the maintainer (please include your contact information).

Please include the following in your report:

Type of issue (e.g., buffer overflow, SQL injection, cross-site scripting)
Full paths of source file(s) related to the manifestation of the issue
The location of the affected code
A proof-of-concept (PoC) or steps to reproduce the issue

We will acknowledge your report within 48 hours and provide a timeline for resolution.
